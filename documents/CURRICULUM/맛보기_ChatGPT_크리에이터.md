## 📖 과정 소개

### 타이틀
"왜 AI 파트너 협업 방식인가?"

### 내용

**ChatGPT는 주니어 3년차 직원입니다**

이 과정에서 ChatGPT는 단순한 도구가 아니라 **함께 일하는 주니어 직원**입니다. 당신은 제작자이자 디렉터로서 AI에게 명확히 지시하고, 결과를 검토하고, 수정을 요청하며, 최종 승인합니다.

- **일을 시킨다**: "웹툰 스토리 8컷을 기획해줘"
- **선택지를 받는다**: "3가지 방향성을 제안해줘"
- **선택하고 지시한다**: "2번 방향으로 가되, 주인공 성격은 더 밝게"
- **체크하고 수정한다**: "대사가 어색해. 10대 말투로 다시"
- **완성한다**: "최종 확인 후 이미지 생성 프롬프트 작성"

**구조적 체이닝 프롬프트**

단발성 질문이 아니라 **단계별로 연결되는 대화 구조**를 설계합니다. 마치 실제 프로젝트처럼 기획 → 초안 → 검토 → 수정 → 완성의 흐름을 만듭니다.

```
1단계: "판타지 웹툰 아이디어 3가지 제안해줘"
2단계: "1번 아이디어로 주인공 캐릭터 만들어줘"
3단계: "이 캐릭터로 첫 에피소드 기획해줘"
4단계: "기획안 검토 - 긴장감 더 필요해"
5단계: "수정된 스토리로 8컷 구성해줘"
```

**장인 정신: 체크 → 성찰 → 수정 → 질문 → 완성**

AI가 만든 결과물을 **무조건 수용하지 않습니다**. 제작자로서 끊임없이 질문하고 개선합니다.

- ✅ "이 대사가 자연스러운가?"
- ✅ "캐릭터 성격과 일치하는가?"
- ✅ "독자가 공감할 수 있는가?"
- ✅ "더 좋은 표현은 없는가?"
- ✅ "전체 스토리와 연결되는가?"

**역공부 방식: 완성품 분석 → 제작 이해**

처음부터 빈 화면에서 시작하지 않습니다. **완성된 웹툰을 먼저 분석**하고, 이것이 어떻게 만들어졌는지 역추적하며 제작 프로세스를 이해합니다.

1. **완성품 감상**: 전문가 수준 웹툰 1화 전체 읽기
2. **구조 분석**: 어떤 프롬프트로 만들어졌을까?
3. **역설계**: 각 단계별 프롬프트 재구성
4. **직접 제작**: 배운 구조로 나만의 콘텐츠 만들기
5. **비교 개선**: 내 작품과 원본 비교하며 개선

**열린 질문으로 AI와 대화하기**

❌ 닫힌 질문: "웹툰 만들어줘" (결과만 받음)
✅ 열린 질문: "웹툰 기획 시 고려할 요소가 뭐야?" (함께 사고)

❌ 명령: "주인공 만들어줘"
✅ 협업: "주인공을 만들고 싶어. 어떤 정보가 필요해?"

**제작자 마인드셋**

당신은 AI를 사용하는 사람이 아니라, **AI를 관리하는 제작자**입니다. 
- 기획은 당신이 합니다
- 방향은 당신이 정합니다
- 최종 판단은 당신이 합니다
- AI는 당신의 파트너입니다

### 이미지 (3개)
- 제작자와 AI가 협업하는 개념도
- 체이닝 프롬프트 구조 다이어그램
- 완성품 분석 → 제작 과정 역추적 흐름도

### 역공부 1단계: 완성품 분석

**먼저 완성된 웹툰 1화를 읽습니다 (제공)**

```
【완성 웹툰: "정의 해커" 1화】
- 8컷 구성
- 주인공: 한지우 (16세, 고1)
- 스토리: 새 학교에서 학교폭력 목격 → 해킹으로 증거 수집
- 클라이맥스: 익명으로 증거 공개
```

**학생 활동: 역설계**

```
질문 1: 이 웹툰을 만들려면 어떤 정보가 필요했을까?
→ 캐릭터 설정, 배경 스토리, 사건 구조

질문 2: 어떤 순서로 프롬프트를 작성했을까?
→ 1) 장르 결정 2) 캐릭터 3) 사건 4) 8컷 구성 5) 대사

질문 3: AI에게 어떻게 일을 시켰을까?
→ 열린 질문으로 선택지 받기 → 선택 → 수정 요청
```

### 체이닝 프롬프트 2단계: 구조적 대화 설계

**단발성 프롬프트 (❌ 나쁜 예)**

```
"학원물 웹툰 만들어줘"
→ 결과: 애매하고 일반적인 스토리
```

**체이닝 프롬프트 (✅ 좋은 예)**

```
━━━━━━━━━━━━━━━━━━━━━━
🔗 1단계: 열린 질문으로 방향 정하기
━━━━━━━━━━━━━━━━━━━━━━

제작자 (당신):
"학원물 웹툰을 만들고 싶어. 
최근 MZ세대가 공감하는 학원물 트렌드가 뭐야?"

AI (주니어):
"현재 학원물 트렌드는 3가지입니다:
1. 현실적 학교폭력 + 정의 구현 (예: 더 글로리)
2. 판타지 능력 + 학원 배경 (예: 나 혼자만 레벨업)
3. 일상 공감 + 힐링 (예: 우리들의 블루스)

어떤 방향을 고려하시나요?"

제작자:
"1번 방향으로 가되, 주인공이 해킹 기술로 증거를 모으는 설정은 어때?"

AI:
"좋습니다! 사이버 정의 구현 콘셉트네요. 
주인공 캐릭터를 구체화하기 전에 몇 가지 질문드립니다:
- 성별과 나이는?
- 해킹을 배운 계기는?
- 주인공의 딜레마(갈등)는 무엇인가요?"

━━━━━━━━━━━━━━━━━━━━━━
🔗 2단계: 선택지 받고 선택하기
━━━━━━━━━━━━━━━━━━━━━━

제작자:
"여자 주인공으로 가고 싶어. 3가지 캐릭터 타입을 제안해줘."

AI:
"【캐릭터 타입 3가지】

A. 조용한 천재형
   - 평소 말수 없음, 코딩 천재
   - 감정 표현 서툼
   - 정의감으로 행동

B. 열혈 정의파
   - 적극적이고 직설적
   - 해킹은 도구일 뿐
   - 사람들과 친밀

C. 이중생활형
   - 겉으로는 평범한 학생
   - 온라인에서는 유명한 해커
   - 정체 감추며 활동

어떤 타입이 스토리에 잘 맞을까요?"

제작자:
"A와 C를 섞고 싶어. 평소 조용하지만 온라인에서 활발한 캐릭터."

AI:
"좋은 선택입니다! 이중 페르소나가 캐릭터에 깊이를 줍니다.
그럼 구체적인 프로필을 작성하겠습니다..."

━━━━━━━━━━━━━━━━━━━━━━
🔗 3단계: 체크 + 성찰 (장인 정신)
━━━━━━━━━━━━━━━━━━━━━━

AI가 캐릭터 프로필을 작성했습니다:

【한지우 프로필】
이름: 한지우 (온라인명: Ghost_J)
나이: 16세 (고1)
성격: 내성적, 정의감, 이중생활
...

제작자 (체크):
"잠깐, 이 캐릭터가 16세 여고생으로 자연스러운가?
해킹 실력이 너무 뛰어난 거 아니야?"

AI:
"좋은 지적입니다. 현실감을 위해 수정하겠습니다:
- 해킹 실력: 전문가급 → 중급 수준으로 하향
- 배운 계기: 천재성 → 중학생 때 온라인 강의로 독학
- 한계 설정: 고난이도 해킹은 실패함, 실수도 함
이렇게 수정하는 게 어떨까요?"

제작자:
"좋아! 실수하는 주인공이 더 공감 가. 수정해줘."

━━━━━━━━━━━━━━━━━━━━━━
🔗 4단계: 반복 수정 (만족할 때까지)
━━━━━━━━━━━━━━━━━━━━━━

### FAQ

**Q1. ChatGPT Playground는 뭐가 다른가요?**
- A: Playground는 **시스템 프롬프트**를 설정할 수 있어, AI의 역할을 명확히 지정할 수 있습니다.
  - 예: "너는 전문 웹툰 작가야" → AI가 웹툰 작가 관점으로 답변
  - 온도(Temperature) 조절로 창의성 제어 가능
  - 대화 히스토리를 저장하여 긴 체이닝 작업 가능

**Q2. 웹툰, 영상, 게임 구조를 꼭 배워야 하나요?**
- A: 네! 각 콘텐츠는 고유한 구조가 있습니다.
  - 웹툰: 8컷 기승전결 + 클리프행어
  - 영상: 3막 구조 (훅-전개-클라이맥스)
  - 게임: 퀘스트 트리 + 대화 분기
  구조를 모르면 AI도 제대로 된 콘텐츠를 만들 수 없습니다.

**Q3. "구조가 뭐야?"라고 물어보면 AI가 가르쳐주나요?**
- A: 네! 열린 질문으로 AI와 함께 배웁니다.
  - "웹툰 8컷 구조가 뭐야?" → AI가 설명
  - "왜 이 구조를 써?" → AI가 이유 설명
  - "다른 구조도 있어?" → AI가 대안 제시

**Q4. AI가 만든 걸 그대로 쓰면 안 되나요?**
- A: 안 됩니다. AI는 주니어 직원입니다. 당신이 최종 승인자입니다.
  - 체크: "이게 자연스러워?"
  - 성찰: "왜 어색하지?"
  - 수정: "~로 다시 작성해줘"

**Q5. 프롬프트를 몇 번이나 써야 하나요?**
- A: 웹툰 1화 기준 20-50개 프롬프트를 체이닝합니다.
  - 단발성 프롬프트(1개)는 저품질
  - 체이닝 프롬프트(20-50개)는 고품질

**Q6. 코딩을 배워야 하나요?**
