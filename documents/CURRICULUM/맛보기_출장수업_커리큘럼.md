## 📱 1과정: 컴퓨터 비전 AI (블록 코딩)

### 벤치마킹 & 기획 배경 (15분)

#### 왜 컴퓨터 비전인가?

**현실 사례 분석:**

**토론 질문 (5분):**
1. "스마트폰에서 컴퓨터 비전을 사용하는 예를 찾아볼까요?"
2. "만약 여러분이 컴퓨터 비전 기술로 무엇을 만들고 싶나요?"
3. "기술이 어렵지 않다면 어떤 프로젝트를 해보고 싶은가요?"

**기획 의도:**
- ❌ 문법 먼저 X → ✅ 프로젝트로 시작
- ❌ 이론 공부 X → ✅ 실행하며 배우기
- ❌ 처음부터 코딩 X → ✅ 이미 만들어진 것 수정

### 2시간 과정: 손 제스처 게임 만들기

#### 타임라인

| 시간 | 내용 | 활동 | 비고 |
|------|------|------|------|
| **0:00-0:15** | 벤치마킹 & 기획 | 토론, 사례 분석 | 발표 중심 |
| **0:15-0:25** | PRIMM: Predict | 완성작 체험 | 게임 플레이 |
| **0:25-0:35** | PRIMM: Run | 코드 실행 | Teachable Machine |
| **0:35-0:50** | PRIMM: Investigate | 알고리즘 탐구 | 핵심 원리 |
| **0:50-1:20** | PRIMM: Modify | 수정하며 배우기 | 단계별 개선 |
| **1:20-1:50** | PRIMM: Make | 나만의 게임 | 창작 시간 |
| **1:50-2:00** | 마무리 | 발표, 확장 가능성 | 공유 |

#### 실습 상세

**1단계: Predict (10분) - 완성된 게임 체험**

**강사:**
"여러분, 이 게임을 한번 해볼까요? 손 모양으로 조종하는 가위바위보 게임이에요!"

*완성된 게임 실행 (DWAI + Teachable Machine)*

**학생 활동:**
- 게임 플레이 (3분)
- 관찰 및 예측:
  - "어떻게 손을 인식할까?"
  - "어떤 기술이 사용됐을까?"
  - "어떻게 만들어졌을까?"

**예측 시트 작성:**
```
1. 손 인식 방법: _________________
2. 게임 진행 순서: _________________
3. 점수 계산 방법: _________________
```

**2단계: Run (10분) - 코드 실행**

**제공 자료:**
- Teachable Machine 학습 완료된 모델
- DWAI 프로젝트 파일
- 단계별 실행 가이드

**강사 시연:**
1. Teachable Machine 모델 로드
2. DWAI에서 블록 확인
3. 웹캠 연결 → 실행

**학생 활동:**
- 제공된 프로젝트 실행
- 자신의 손으로 테스트
- 인식 확인

**3단계: Investigate (15분) - 알고리즘 탐구**

**핵심 알고리즘 설명:**

**주요 알고리즘 3가지:**

1. **이미지 분류 알고리즘**
```
입력: 웹캠 이미지
처리: AI 모델이 학습한 패턴과 비교
출력: 각 클래스별 확률 (바위 80%, 보 15%, 가위 5%)
판단: 가장 높은 확률을 선택
```

2. **게임 로직 알고리즘**
```
IF 플레이어 = 바위 AND 컴퓨터 = 가위:
    플레이어 승리, +10점
ELSE IF 플레이어 = 바위 AND 컴퓨터 = 보:
    플레이어 패배, -5점
ELSE IF 플레이어 = 바위 AND 컴퓨터 = 바위:
    무승부, 0점
```

3. **랜덤 선택 알고리즘**
```
random_number = 랜덤(1~3)
IF random_number = 1: 컴퓨터 = 바위
IF random_number = 2: 컴퓨터 = 보
IF random_number = 3: 컴퓨터 = 가위
```

**탐구 활동:**
- 블록 코드 읽어보기
- 각 블록의 역할 확인
- 흐름도 그려보기

**4단계: Modify (30분) - 단계별 수정**

**수정 미션 1 (10분): 점수 시스템 변경**
```
현재: 승리 +10점, 패배 -5점
미션: 승리 +20점, 패배 -10점으로 변경하기

힌트: "점수" 블록 찾아서 숫자만 바꾸면 됩니다!
```

**수정 미션 2 (10분): 효과음 추가**
```
미션: 승리/패배/무승부 시 각각 다른 소리 나게 하기

힌트: DWAI의 "소리 재생" 블록 사용
```

**수정 미션 3 (10분): 승부 방식 변경**
```
현재: 바로 승부
미션: "3, 2, 1" 카운트다운 후 승부

힌트: "기다리기" 블록 + "말하기" 블록 조합
```

**5단계: Make (30분) - 나만의 게임**

**창작 옵션 선택:**

**옵션 A: 손가락 카운팅 게임**
- 화면에 숫자 표시 (1~5)
- 손가락으로 해당 숫자 표현
- 빠르게 맞출수록 높은 점수

**옵션 B: 손 모양 퀴즈**
- AI가 손 모양 제시
- 학생이 따라하기
- 정확도 평가

**옵션 C: 자유 주제**
- 학생 아이디어로 새로운 게임

**지원 사항:**
- Teachable Machine으로 새로운 손 모양 학습
- 기존 블록 재사용
- 강사 실시간 도움

**6단계: 마무리 (10분)**

**발표 (5분):**
- 2-3명 자원자 발표
- 어떤 부분을 수정했는지
- 어려웠던 점

**확장 가능성 (5분):**
1. 얼굴 인식으로 확장 → 표정 게임
2. 사물 인식으로 확장 → 물체 분류 게임
3. 정식 과정에서 배울 내용 소개

### 2시간 과정: AI로 유튜브 쇼츠 대본 만들기

#### 타임라인

| 시간 | 내용 | 활동 |
|------|------|------|
| **0:00-0:15** | 벤치마킹 & 기획 | 토론 |
| **0:15-0:25** | PRIMM: Predict | 완성 쇼츠 시청 |
| **0:25-0:35** | PRIMM: Run | ChatGPT 실행 |
| **0:35-0:50** | PRIMM: Investigate | 프롬프트 분석 |
| **0:50-1:30** | PRIMM: Modify | 대본 개선 |
| **1:30-1:50** | PRIMM: Make | 나만의 쇼츠 |
| **1:50-2:00** | 마무리 | 발표 |

#### 실습 상세

**1단계: Predict (10분) - 완성 쇼츠 분석**

**강사:**
"이 60초 쇼츠를 보고, 어떻게 만들어졌을지 예측해볼까요?"

*AI로 만든 쇼츠 3개 시청*

**학생 활동:**
- 쇼츠 구조 분석
- 대본 패턴 찾기
- 후크, 전개, 마무리 확인

**분석 시트:**
```
쇼츠 제목: _________________
후크 (0-3초): _________________
핵심 내용 (3-50초): _________________
CTA (50-60초): _________________
예상 제작 시간: _________________
```

**2단계: Run (10분) - ChatGPT 실행**

**제공 자료:**
- ChatGPT 프롬프트 템플릿
- 예시 대본 3개
- 단계별 가이드

**강사 시연:**
```
학생 → ChatGPT:

"60초 유튜브 쇼츠 대본을 작성해줘.

주제: 아침 루틴
타겟: 10대
톤: 에너지 넘치는

구조:
- 0-3초: 강력한 후크
- 3-50초: 5가지 루틴
- 50-60초: 구독 유도"
```

**학생 활동:**
- 프롬프트 입력
- ChatGPT 응답 확인
- 대본 읽어보기

**3단계: Investigate (15분) - 프롬프트 분석**

**핵심 알고리즘: 효과적인 프롬프트 구조**

**프롬프트 엔지니어링 3대 원칙:**

**원칙 1: 명확성**
```
❌ 나쁜 예:
"유튜브 대본 써줘"

✅ 좋은 예:
"60초 유튜브 쇼츠 대본 작성
주제: 효율적인 공부법
타겟: 중학생
구조: 후크(3초) + 5가지 팁(50초) + CTA(7초)"
```

**원칙 2: 구조화**
```
✅ 템플릿 사용:
"다음 형식으로 작성해줘:

[0-3초] 후크: _______
[3-15초] 팁 1: _______
[15-27초] 팁 2: _______
..."
```

**원칙 3: 반복 개선**
```
1차 생성 → 검토 → 개선 요청

"후크를 더 임팩트 있게 만들어줘"
"팁을 더 구체적으로 설명해줘"
"CTA를 강력하게 바꿔줘"
```

**4단계: Modify (40분) - 단계별 개선**

**수정 미션 1 (10분): 후크 개선**
```
현재 후크: "공부 잘하는 법 알려드릴게요"

개선 요청:
"후크를 더 임팩트 있게 만들어줘.
- 질문으로 시작
- 호기심 자극
- 3초 안에 집중"

예상 결과:
"성적 1등의 비밀, 알고 싶어?"
```

**수정 미션 2 (15분): 내용 구체화**
```
현재: 추상적인 팁
개선: 실행 가능한 구체적 팁

프롬프트:
"각 팁을 더 구체적으로 만들어줘:
- 숫자 포함
- 실행 단계 명시
- 예시 추가"
```

**수정 미션 3 (15분): 개인화**
```
미션: 나만의 스타일로 변경

프롬프트 예시:
"이 대본을 다음 스타일로 변경:
- 톤: [재미있는/진지한/동기부여]
- 말투: [반말/존댓말/친근한]
- 특징: [이모티콘 사용/전문용어/유머]"
```

**5단계: Make (20분) - 나만의 쇼츠**

**주제 선택:**
- 학교 생활
- 취미/관심사
- 꿀팁 공유
- 챌린지

**제작 과정:**

1. **아이디어 브레인스토밍 (5분)**
```
프롬프트:
"[선택한 주제]로 유튜브 쇼츠 아이디어 10개 제안해줘.
각 아이디어는:
- 제목
- 후크
- 핵심 메시지
- 예상 조회수 범위"
```

2. **대본 작성 (10분)**
```
선택한 아이디어로 완성 대본 작성
+ ChatGPT와 3회 이상 대화로 개선
```

3. **비주얼 가이드 (5분)**

```cpp
#include <WiFi.h>

// 핀 설정
#define LED_PIN 2
#define TRIG_PIN 5
#define ECHO_PIN 18

// WiFi 설정
const char* ssid = "WiFi_이름";
const char* password = "비밀번호";

// LED 상태
bool ledOn = false;

void setup() {
  Serial.begin(115200);
  pinMode(LED_PIN, OUTPUT);
  pinMode(TRIG_PIN, OUTPUT);
  pinMode(ECHO_PIN, INPUT);
  
  // WiFi 연결
  WiFi.begin(ssid, password);
  while (WiFi.status() != WL_CONNECTED) {
    delay(500);
    Serial.print(".");
  }
  Serial.println("\nWiFi 연결 완료!");
}

void loop() {
  int distance = getDistance();
  
  // 30cm 이내 손 감지
  if (distance < 30) {
    ledOn = !ledOn;  // 토글
    digitalWrite(LED_PIN, ledOn);
    Serial.println(ledOn ? "LED ON" : "LED OFF");
    delay(500);  // 디바운스
  }
  
  delay(100);
}

int getDistance() {
  digitalWrite(TRIG_PIN, LOW);
  delayMicroseconds(2);
  digitalWrite(TRIG_PIN, HIGH);
  delayMicroseconds(10);
  digitalWrite(TRIG_PIN, LOW);
  
  long duration = pulseIn(ECHO_PIN, HIGH);
  return duration * 0.034 / 2;
}
```

**거리 계산 공식:**
```
초음파 속도: 340 m/s = 0.034 cm/μs

거리 (cm) = 왕복 시간(μs) × 0.034 / 2

예시:
왕복 시간 = 1000μs
거리 = 1000 × 0.034 / 2 = 17cm
```

```
       return  // 너무 멀면 무시
   
3. LED 상태 토글
   ledOn = !ledOn  // true → false, false → true
   
4. LED 제어

5. 디바운스
```

```cpp
#include <DHT.h>

#define DHT_PIN 4
DHT dht(DHT_PIN, DHT22);

void setup() {

  dht.begin();
}

void loop() {
  float temp = dht.readTemperature();
  float humi = dht.readHumidity();
  
  // Early return: 센서 오류
  if (isnan(temp) || isnan(humi)) {
    Serial.println("센서 오류!");
    delay(2000);
    return;
  }
  
  Serial.printf("온도: %.1f°C, 습도: %.1f%%\n", temp, humi);
  
  // 알림 로직
  if (temp > 28) {
    Serial.println("⚠️ 온도가 높습니다!");
  }
  if (humi < 30) {
    Serial.println("⚠️ 습도가 낮습니다!");
  }
  
  delay(2000);
}
```

**제공 코드:**
```cpp
#define PIR_PIN 5
#define LED_PIN 2
#define BUZZER_PIN 15

bool alarmActive = false;

void setup() {
  Serial.begin(115200);
  pinMode(PIR_PIN, INPUT);
  pinMode(LED_PIN, OUTPUT);
  pinMode(BUZZER_PIN, OUTPUT);
}

void loop() {
  int motion = digitalRead(PIR_PIN);
  
  if (motion == HIGH && !alarmActive) {
    Serial.println("🚨 모션 감지!");
    triggerAlarm();
    delay(2000);  // 재감지 방지
  }
  
  delay(100);
}

void triggerAlarm() {
  // LED 깜빡임
  for (int i = 0; i < 5; i++) {
    digitalWrite(LED_PIN, HIGH);
    tone(BUZZER_PIN, 1000);
    delay(200);
    digitalWrite(LED_PIN, LOW);
    noTone(BUZZER_PIN);
    delay(200);
  }
}
```

**Investigate (10min):**

**PIR 센서 원리:**
```
수동형 적외선 (PIR):

1. 적외선 감지
   - 모든 물체는 적외선 방출
   - 사람의 체온 (36.5°C) 감지
   
2. 변화 감지
   - 2개의 센서로 분할 감지
   - 센서 간 차이 발생 시 모션 감지
   
3. 감지 범위
   - 거리: 5-7m
   - 각도: 120°
   - 감지 지연: 2-3초
```

**미션 2: 스마트 조명 통합**

