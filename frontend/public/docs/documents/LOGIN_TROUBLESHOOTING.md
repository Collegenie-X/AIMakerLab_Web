# 로그인 문제 해결 가이드

## 문제: "이메일 또는 비밀번호가 올바르지 않습니다" 에러

### 원인

데모 환경에서는 회원가입 시 이메일 인증이 필요했었습니다. 이전에 회원가입한 계정은 인증되지 않은 상태일 수 있습니다.

### 해결 방법

#### 방법 1: 새로 회원가입하기 (가장 간단)

1. **로그인 다이얼로그**를 엽니다
2. **"회원가입"** 탭으로 전환합니다
3. 다른 이메일로 새로 가입합니다
4. 이제 **자동으로 이메일 인증이 완료**됩니다
5. 바로 로그인할 수 있습니다

#### 방법 2: 기존 데이터 초기화하기

브라우저 개발자 도구를 사용하여 LocalStorage를 초기화할 수 있습니다.

**Step 1: 개발자 도구 열기**
- Windows/Linux: `F12` 또는 `Ctrl + Shift + I`
- Mac: `Cmd + Option + I`

**Step 2: Console 탭으로 이동**

**Step 3: 다음 명령어 입력**

```javascript
clearAllUsers()
```

**Step 4: 페이지 새로고침**
- Windows/Linux: `Ctrl + R`
- Mac: `Cmd + R`

**Step 5: 다시 회원가입**
- 이제 같은 이메일로 다시 가입할 수 있습니다
- 자동으로 이메일 인증이 완료됩니다

#### 방법 3: LocalStorage 수동 삭제

**Step 1: 개발자 도구 열기**

**Step 2: Application (또는 Storage) 탭으로 이동**

**Step 3: Local Storage 선택**
- 왼쪽 사이드바에서 `Local Storage` 확장
- 현재 사이트 URL 클릭

**Step 4: 다음 항목들 삭제**
- `demo_users`
- `current_user_email`

**Step 5: 페이지 새로고침 후 다시 회원가입**

## 변경 사항 (2024-11-17 업데이트)

### ✅ 개선된 사항

1. **자동 이메일 인증**
   - 회원가입 시 이메일 인증이 자동으로 완료됩니다
   - 별도의 인증 링크 클릭이 필요 없습니다

2. **명확한 에러 메시지**
   - 이메일 인증이 필요한 경우 명확한 안내 메시지 표시
   - "이메일 인증이 필요합니다. 데모 환경에서는 회원가입을 다시 해주세요."

3. **개발자 도구 함수**
   - 브라우저 콘솔에서 `clearAllUsers()` 함수로 간편하게 초기화 가능

## 데모 환경 특징

현재 인증 시스템은 **데모 환경용**입니다:

- ✅ LocalStorage를 사용한 클라이언트 측 저장
- ✅ 비밀번호 평문 저장 (실제 서비스에서는 해싱 필요)
- ✅ 자동 이메일 인증
- ✅ 브라우저 콘솔에서 데이터 관리 가능

## 프로덕션 환경으로 전환 시

실제 서비스에서는 다음과 같이 변경해야 합니다:

1. **백엔드 API 연동**
   - Django REST Framework와 연결
   - JWT 또는 Session 기반 인증 구현

2. **이메일 인증**
   - 실제 이메일 발송 서비스 연동
   - SendGrid, AWS SES 등 사용

3. **비밀번호 보안**
   - 백엔드에서 bcrypt 등으로 해싱
   - HTTPS 필수

4. **토큰 관리**
   - HttpOnly 쿠키 사용
   - CSRF 토큰 구현

## 테스트 계정 생성하기

### 빠른 테스트

1. 로그인 버튼 클릭
2. "회원가입" 탭 선택
3. 테스트 이메일 입력 (예: `test@example.com`)
4. 비밀번호 입력 (최소 6자)
5. 필수 약관 동의
6. "회원가입" 버튼 클릭
7. "회원가입이 완료되었습니다!" 메시지 확인
8. 자동으로 "로그인" 탭으로 전환됨
9. 같은 이메일/비밀번호로 로그인

### 주의사항

- 데모 환경이므로 페이지를 새로고침하면 LocalStorage 데이터는 유지되지만, 브라우저를 닫으면 데이터가 남아있습니다
- 시크릿/프라이빗 모드에서는 브라우저를 닫으면 데이터가 삭제됩니다

## 문제가 계속되는 경우

1. **브라우저 캐시 삭제**
   - Windows/Linux: `Ctrl + Shift + Delete`
   - Mac: `Cmd + Shift + Delete`

2. **다른 브라우저에서 시도**
   - Chrome, Firefox, Safari 등

3. **콘솔 에러 확인**
   - 개발자 도구 Console 탭에서 에러 메시지 확인
   - 에러가 있다면 개발팀에 공유
