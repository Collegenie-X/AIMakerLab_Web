# 사용자 페이지 가이드 (Udemy/쇼핑몰 스타일)

## 개요

일반적인 쇼핑몰/Udemy 스타일의 사용자 페이지로 구현되었습니다. 
**헤더, 푸터가 있는 일반 페이지** 구조로, 로그인한 사용자가 자신의 정보를 **조회(Read)**할 수 있습니다.

## 📊 페이지 구조 비교

### ❌ 이전 구조 (관리자 스타일)
```
/dashboard (사이드바가 있는 별도 레이아웃)
  ├─ /courses
  ├─ /comments
  ├─ /gallery
  └─ /profile
```
- 사이드바 고정
- 헤더/푸터 없음
- 관리자 대시보드 느낌

### ✅ 현재 구조 (Udemy/쇼핑몰 스타일)
```
헤더 (로그인 시: 사용자메일▼)
  └─ 나의 강의 → /my-courses
  └─ 프로필 설정 → /my-profile
  └─ 로그아웃

각 페이지:
  Header
  ─────────
  콘텐츠
  ─────────
  Footer
```

## 🎯 페이지 구성

### 1. `/my-courses` - 나의 강의

**위치**: 헤더 > 사용자 이메일 클릭 > "나의 강의"

**기능**:
- ✅ 수강 중인 강의 목록 조회
- ✅ 강의 상태 표시 (진행중/예정/완료)
- ✅ 진행률 표시
- ✅ 강의 상세보기 (강의 페이지로 이동)

**레이아웃**:
```
┌────────────────────────────────┐
│         Header                 │
├────────────────────────────────┤
│                                │
│  나의 강의                      │
│  수강 중인 강의 목록입니다       │
│                                │
│  ┌─────┐  ┌─────┐  ┌─────┐   │
│  │강의1│  │강의2│  │강의3│   │
│  │진행률│  │진행률│  │진행률│   │
│  └─────┘  └─────┘  └─────┘   │
│                                │
├────────────────────────────────┤
│         Footer                 │
└────────────────────────────────┘
```

**주요 정보**:
- 강의 제목
- 강사명
- 시작일/종료일
- 진행률 (진행 중인 강의만)
- 카테고리
- 상태 배지

**액션**:
- 강의 보기 버튼 클릭 → 해당 강의 페이지로 이동

### 2. `/my-profile` - 프로필 설정

**위치**: 헤더 > 사용자 이메일 클릭 > "프로필 설정"

**기능**:
- ✅ 프로필 정보 조회/수정
- ✅ 비밀번호 변경
- ✅ 탭으로 구분된 UI

**레이아웃**:
```
┌────────────────────────────────┐
│         Header                 │
├────────────────────────────────┤
│                                │
│  프로필 설정                     │
│                                │
│  [ 프로필 정보 ][ 보안 설정 ]   │
│  ─────────────────────────     │
│                                │
│  이름:    [입력]                │
│  이메일:  [email@example.com]   │
│  연락처:  [입력]                │
│  주소:    [입력]                │
│                                │
│  [프로필 업데이트]              │
│                                │
├────────────────────────────────┤
│         Footer                 │
└────────────────────────────────┘
```

**프로필 정보 탭**:
- 이름 (수정 가능)
- 이메일 (읽기 전용)
- 연락처 (수정 가능)
- 주소 (수정 가능)

**보안 설정 탭**:
- 현재 비밀번호 입력
- 새 비밀번호 입력
- 새 비밀번호 확인
- 비밀번호 표시/숨김 토글

## 📱 반응형 디자인

### 데스크톱 (1024px+)
- 헤더에 사용자 이메일 + 드롭다운 메뉴
- 3단 그리드로 강의 표시

### 태블릿 (768px~1023px)
- 헤더에 사용자 이메일 + 드롭다운 메뉴
- 2단 그리드로 강의 표시

### 모바일 (~767px)
- 햄버거 메뉴
- "내 학습" 섹션 표시
  - 나의 강의
  - 프로필 설정
- 1단 그리드로 강의 표시

## 🚀 사용자 플로우

### 로그인 전
```
1. 홈페이지 방문
2. 헤더에서 "로그인" 버튼 클릭
3. 이메일/비밀번호 입력
4. 로그인 성공 → 페이지 새로고침
```

### 로그인 후 - 나의 강의 확인
```
1. 헤더에서 자신의 이메일 클릭
2. 드롭다운 메뉴에서 "나의 강의" 선택
3. /my-courses 페이지로 이동
4. 수강 중인 강의 목록 확인
5. "강의 보기" 버튼 클릭
6. 해당 강의 상세 페이지로 이동
```

### 로그인 후 - 프로필 수정
```
1. 헤더에서 자신의 이메일 클릭
2. 드롭다운 메뉴에서 "프로필 설정" 선택
3. /my-profile 페이지로 이동
4. "프로필 정보" 탭에서 정보 수정
5. "프로필 업데이트" 버튼 클릭
6. 성공 메시지 확인
```

### 로그인 후 - 비밀번호 변경
```
1. 프로필 설정 페이지로 이동
2. "보안 설정" 탭 클릭
3. 현재 비밀번호 입력
4. 새 비밀번호 입력 (6자 이상)
5. 새 비밀번호 확인 입력
6. "비밀번호 변경" 버튼 클릭
7. 성공 메시지 확인
```

## 🎯 주요 특징

### 1. **Read-Only 중심**
- CRUD가 아닌 **Read(조회) 중심**
- 프로필 수정과 비밀번호 변경만 가능
- 강의는 조회만 가능, 클릭 시 상세 페이지로 이동

### 2. **일관된 레이아웃**
```typescript
// 모든 페이지가 동일한 구조
<div className="flex min-h-screen flex-col">
  <Header />
  <main className="flex-1 bg-gray-50">
    {/* 콘텐츠 */}
  </main>
  <Footer />
</div>
```

### 3. **인증 체크**
```typescript
const { userEmail } = useAuthGuard()

if (!userEmail) {
  // 로그인 안내 화면 표시
  return <LoginPrompt />
}
```

### 2. **Django REST Framework 연동**
```python
# backend/accounts/views.py
@api_view(['GET'])
@permission_classes([IsAuthenticated])
def my_courses(request):
    """사용자의 강의 목록"""
    courses = Course.objects.filter(students=request.user)
    serializer = CourseSerializer(courses, many=True)
    return Response(serializer.data)
```

## 🆚 /dashboard vs /my-courses 차이점

| 항목 | /dashboard (관리자 스타일) | /my-courses (일반 사용자) |
|------|--------------------------|-------------------------|
| 레이아웃 | 사이드바 고정 | 헤더/푸터 |
| 헤더 | ❌ 없음 | ✅ 있음 |
| 푸터 | ❌ 없음 | ✅ 있음 |
| 용도 | 관리자/운영자 | 일반 사용자 |
| 네비게이션 | 사이드바 메뉴 | 헤더 드롭다운 |
| 느낌 | 대시보드/관리 도구 | 쇼핑몰/학습 사이트 |
